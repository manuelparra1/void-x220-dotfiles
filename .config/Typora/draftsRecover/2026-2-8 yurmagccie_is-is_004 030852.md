**IS-IS. Part 4. Wide-metrics**

**IS Reachability TLV #2** – The TLV conveys pure router-to-router connectivity information and is unrelated to IP. Metric information in this TLV is coded with 6-bit value, which results in possible values from 0 to 63. The reason for such small value is a CPU and RAM shortage issue of the time.

**Internal IP Reachability TLV #128**. This TLV is used for describing directly connected prefixes.

**External IP Reachability TLV #130**. This TLV is used for any prefixes that origin from other protocols, like static, BGP, OSPF, RIP.

Later, instead of TLVs #2, #128 and #130 were proposed the following two TLVs: 
– **Extended IS Reachability TLV #22** (the replacement for TLV #2) 
– **Extended IP Reachability TLV #135** (the replacement for TLVs #128 and #130)

In these TLVs size of the metric filed is **24 bits**. These metrics are also known as **wide-metrics.**

Another of the worth noting changes is that TLV #135 has **no internal/external differentiation**in the TLV anymore.

From [The Complete IS-IS Routing Protocol](https://www.springer.com/gp/book/9781852338220) by Hannes Gredler and Walter Goralski:

_Henk Smit, one of the authors of RFC 3784, noted once on a posting to the ISIS-WG mailing list: 
I have never really understood the need for both TLV128 and TLV130. This was also one of the reasons why we decided to not carry the I/E metric bit semantics forward into TLV135 (the new IP prefix TLV defined in the IS-IS TE extensions draft). The most important reason was of course the fact that we had no bits left, and did not want to spend another byte per prefix._

In IOS, you can enable wide-metrics with this command:

```
router isis 1
 metric-style wide
```

In order to enable both old and new-style metrics use **metric-style wide transition** command:

```
router isis 1
 metric-style wide transition
```
